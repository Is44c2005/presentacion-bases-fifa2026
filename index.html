⚠️ El Problema

  • ¿Cómo gestionar 104 partidos, 48 selecciones y 1.248 jugadores con integridad y sin redundancia?
  • Relaciones N:M con semántica propia: cada partido vincula dos selecciones con roles y estadísticas distintas — exige una entidad asociativa para calcular bien la tabla de posiciones
  • Claves compuestas en entidades débiles: Jugador se identifica por numCamiseta + idSeleccion; una FK mal modelada genera huérfanos en la carga masiva
  • Reglas de negocio variables por fase: grupos acumulan puntos, eliminatorias aplican prórroga — sin objetos programables habría que reescribir lógica en cada reporte
  • La solución: un modelo relacional en 3FN en SQL Server con 10 objetos programables

🔧 Procesos Principales

  • Gestión de partidos y resultados oficiales
  • Registro de jugadores con datos reales (FIFA)
  • Cuartetos arbitrales con jerarquía interna
  • Posiciones grupales y tabla dinámica
  • Registro de hinchas por partido
  • Control de estadios por ciudad y capacidad
Objetivo General y Alcance
MundialFIFA2026 · SQL Server · 3FN
Objetivo

Diseñar e implementar una base de datos relacional completa en SQL Server que modele, almacene y consulte toda la información del torneo FIFA World Cup 2026, aplicando normalización hasta Tercera Forma Normal (3FN) y objetos programables para consultas complejas.
